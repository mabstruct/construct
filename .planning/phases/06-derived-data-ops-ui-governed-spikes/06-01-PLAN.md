<objective>
Create stable derived-data contracts for all views JSON files — JSON Schema declarations, version fields in metadata, and Pydantic-backed contract tests that validate format on load.

**Purpose:** ADV-03 requires that the system "emit stable derived-data contracts that a future UI layer can consume without redefining the knowledge model." Without schemas, consumers must reverse-engineer the data format. With schemas and version fields, breaking changes are detectable, backward compatibility is testable, and the views API is self-documenting.

**Output:**
- `src/construct/views/` package with Pydantic model contracts for all 8 views data files
- JSON Schema generated from Pydantic models via `model_json_schema()`
- Schema validation BEFORE write in the views data generator
- Contract tests proving schema conformance on the `test-ws/my-construct/` fixture
</objective>

```python
SCHEMA_VERSION = "0.2.0"

def wrap(data: dict, generated_at: str, build_id: str, workspace_id: str | None = None) -> dict:
    """Wrap each schema's data in the standard envelope."""
    env = {
        "schema_version": SCHEMA_VERSION,
        "generated_at": generated_at,
        "build_id": build_id,
        "data": data,
    }
    if workspace_id is not None:
        env["workspace_id"] = workspace_id
    return env
```
