**`bridge_detect(workspace_path: str) -> OperationResult`** — Main entry point following PIPE handler pattern. Structure:

```python
def bridge_detect(workspace_path: str) -> OperationResult:
    """Run L1→L2→L3 bridge detection pipeline.

    Args:
        workspace_path: Path to CONSTRUCT workspace directory.

    Returns:
        OperationResult with data containing bridges, candidates, and scoring metadata.
    """
    try:
        root = Path(workspace_path)
        loader = WorkspaceLoader(root)

        # L1: Structural edges from connections.json
        l1_results = _l1_structural(loader)

        # L2: Category overlap across domains
        l2_results = _l2_category_overlap(loader)

        # Merge L1 + L2 into candidate list
        candidates = _merge_candidates(l1_results, l2_results, loader)

        # L3: Semantic assessment (only for candidates above threshold)
        l3_results = _l3_semantic(loader, candidates) if candidates else {}

        # Compute scores per spec
        bridges = _compute_all_scores(candidates, l3_results)

        # Persist to log/bridge-candidates.json
        _persist_candidates(root, bridges)

        total_confirmed = bridges["summary"]["totals"]["confirmed"]
        total_strong = bridges["summary"]["totals"]["strong_candidates"]

        return OperationResult(
            success=True,
            message=f"Bridge detection complete: {total_confirmed} confirmed, "
                    f"{total_strong} strong candidates",
            data=bridges,
        )
    except (WorkspaceLoadError, OSError) as exc:
        return OperationResult(
            success=False,
            message=str(exc),
            errors=[OperationError(reason=str(exc))],
        )
```

**`_l1_structural(loader: WorkspaceLoader) -> dict[str, Any]`**:

Load `connections.json` via `loader.load_connections()`. Group connections by domain-pair. For each connection where `from_domain != to_domain`, record it as a structural bridge. Return dict keyed by `"{from_id}--{to_id}"` with connection metadata.

Domain resolution: For each card ID in a cross-domain connection, look up the card's domain by reading its file or using `loader.load_cards()`. For efficiency, pre-load all cards once.

Implementation detail: The WorkspaceLoader's `load_connections()` returns `ConnectionsSchema` with `.connections` list where each connection has `from`, `to`, `type`, `created_by`, `notes`. Map these to card IDs, then look up card domains from loaded cards.

**`_l2_category_overlap(loader: WorkspaceLoader) -> dict[str, Any]`**:

Load all cards via `loader.load_cards()`. Group cards by domain. For each domain pair, compute the intersection of their `content_categories`. If intersection has >= 1 shared category, record as candidate. Return dict keyed by domain-pair with shared categories list.

Important: Domain self-pairs (same domain) should be excluded. Only cross-domain pairs.

**`_merge_candidates(l1_results: dict, l2_results: dict, loader: WorkspaceLoader) -> list[dict]`**:

Merge L1 structural edges and L2 category overlaps into a unified candidate list. Each candidate has: `from_card_id`, `to_card_id`, `from_domain`, `to_domain`, `l1_structural` (bool), `l2_shared_categories` (list[str]), and a combined L1+L2 `pre_score`.

Pre-score formula: `(L1_WEIGHT * (1.0 if l1 else 0.0)) + (L2_WEIGHT * min(1.0, len(shared_categories) / 3.0))`

**`_l3_semantic(loader: WorkspaceLoader, candidates: list[dict]) -> dict[str, Any]`**:

For each candidate with `pre_score >= L3_THRESHOLD`:
1. Load up to 300 chars from each card's body
2. Call LLM with prompt asking to classify as "strong" / "possible" / "reject"
3. Record verdict and one-sentence reasoning

Cost guard: If `len(qualifying_candidates) > MAX_L3_CANDIDATES` (50), skip L3 entirely with a warning. Return L1+L2-only results.

L3 implementation uses `ChatAnthropic.with_structured_output(method="json_schema")` with a `BridgeAssessmentOutput` Pydantic model (verdict: "strong" | "possible" | "reject", reasoning: str). Temperature 0.0 for deterministic classification. Use haiku model (workhorse tier) per AI-SPEC §4 Model Configuration table.

Note: Since this is an LLM call, it depends on `langchain_anthropic` (installed in Plan 01). If the API key is unavailable, L3 gracefully degrades — log a warning and return L1+L2-only results with a flag `l3_skipped: true`.

**`_compute_all_scores(candidates: list[dict], l3_results: dict) -> dict[str, Any]`**:

For each candidate:
1. Get L3 verdict from `l3_results` keyed by candidate pair ID
2. Compute final score per spec-v02-cross-domain-data.md §7
3. Assign band

Compute L3 score: "strong" → 1.0, "possible" → 0.6, "reject"/None → 0.0

Band assignment (same as test_bridge_scoring.py):
- l1_structural == True → "strong" (confirmed bridge)
- score >= 0.70 → "strong"
- score >= 0.45 → "medium"  
- else → "weak"

```python
def _persist_candidates(root: Path, bridges: dict) -> None:
    """Write bridge candidates to workspace log and views build directory."""
    serialized = json.dumps(bridges, indent=2, default=str) + "\n"

    # Write to pipeline log
    log_dir = root / "log"
    log_dir.mkdir(parents=True, exist_ok=True)
    (log_dir / "bridge-candidates.json").write_text(serialized, encoding="utf-8")

    # Write to views build data (D-06 #4 — derived data contract)
    views_dir = root / "views" / "build" / "data"
    views_dir.mkdir(parents=True, exist_ok=True)
    (views_dir / "bridges.json").write_text(serialized, encoding="utf-8")
```

| Boundary | Description |
|----------|-------------|
| Card content → LLM (L3) | Card titles and excerpts from different domains sent to LLM for semantic assessment. Only cards from promising L1+L2 candidates. |
| Filesystem → pipeline | Workspace files (connections.json, cards/*.md, domains.yaml) read by WorkspaceLoader. Pipeline never writes to canonical locations (only derived data to log/). |
| Pipeline output → JSON | Bridge candidates written to log/bridge-candidates.json. Derived data only — no canonical SOT modification. |
