Create `src/construct/llm/ask_domain.py` containing:

1. **`AskDomainState`** — TypedDict graph state schema (NOT BaseModel — per LangGraph convention, avoids type coercion issues documented in AI-SPEC §3 Pitfall 2). Fields:
   - Input: `question: str`, `domain_id: str`, `workspace_path: str`, `max_cards: int`
   - Processing: `cards: list[dict]`, `context: str`, `provider: str`, `model: str`
   - LLM output: `synthesised_answer: str | None`, `cited_card_ids: list[str]`, `llm_confidence: str | None`
   - Output: `citations: list[dict]`, `retrieval_cards_considered: int`, `retrieval_cards_selected: int`
   - Metadata: `token_usage: dict | None`, `review_required: bool`, `review_status: str`

2. **`SynthesisOutput`** — Pydantic BaseModel for `with_structured_output(method="json_schema")`:
   - `answer: str` — answer text
   - `cited_card_ids: list[str]` — card IDs from context
   - `confidence: str` — "high" / "medium" / "low"

3. **Six node functions** (each takes `state: AskDomainState` → returns `dict` of state updates):
   - **`load_domain_cards(state)`** — Uses `WorkspaceLoader(state["workspace_path"])` to load all cards (`loader.load_cards()`). Filters out archived (lifecycle != "archived"). Sets `cards` and `retrieval_cards_considered`.
   - **`filter_by_domain(state)`** — Filters `state["cards"]` to only those where `str(c.get("domain_id", "")) == state["domain_id"]`. Returns filtered `cards` list.
   - **`rank_by_relevance(state)`** — Keyword rank: split `state["question"]` into tokens, score each card by token overlap with `(card.get("title","") + " " + card.get("body","")).lower()`. Sort descending by score, take top `state["max_cards"]`. Sets `cards` and `retrieval_cards_selected`.
   - **`build_context(state)`** — Format each card as `"--- Card: {id} ---\nTitle: {title}\nConfidence: {confidence}\nContent: {body[:2000]}"`. Join with newlines into `context`. Hard truncation: if estimated tokens (~4 chars/token) > 100K, drop lowest-ranked cards iteratively.
   - **`llm_synthesize(state)`** — Per AI-SPEC §4b structured output pattern:
     ```python
     llm = ChatAnthropic(model=state.get("model", "claude-sonnet-4-20250514"), temperature=0.2, max_tokens=4096)
     structured_llm = llm.with_structured_output(SynthesisOutput, method="json_schema")
     ```
     System message instructs: answer using ONLY provided context, cite specific card IDs, hedge when confidence < 3. User message: `"Question: {q}\n\nContext:\n{context}"`. Wrap in try/except: on failure return `synthesised_answer=None` with error in `token_usage`.
   - **`extract_citations(state)`** — Map cited card IDs to citation objects with: `card_id`, `title`, `snippet` (body[:300]), `confidence` (source card frontmatter confidence). If a cited ID is not in the retrieved cards set, emit warning via `print()` and drop it. Sets `citations`, `review_required=True`, `review_status="pending"`.

4. **`build_ask_domain_graph() -> StateGraph`** — Construct and compile the graph with linear topology:
   ```
   START → load_domain_cards → filter_by_domain → rank_by_relevance → build_context → llm_synthesize → extract_citations → END
   ```

5. **Empty context guard** — In `run_gate()`, after building context, if `context` is empty (no relevant cards), skip `llm_synthesize` entirely (save LLM call), return structured "unanswerable" response.

6. **`run_gate(gate_id: str, input_data: AskDomainInput, *, config_path: Path | None = None) -> GateResult`** — Loads config via `load_llm_config()`, resolves provider/gate config, builds graph, invokes it, returns `AskDomainOutput` (defined in catalog.py or ask_domain.py) with: `answer`, `citations[]`, `gate_metadata`, `retrieval`, `token_usage`.

7. **Exact-match cache** — A module-level `_CACHE: dict[str, dict]` storing results keyed by `f"{domain_id}::{question}"` with timestamp. If cache hit and < 24h old, return cached result. Per AI-SPEC §4b cost control.

The full code is specified in AI-SPEC.md §4 "Core Pattern — 6-Node Linear StateGraph for ask.domain" (lines 322–543). Use that as the authoritative implementation reference.

def domain(
    ctx: typer.Context,
    question: str = typer.Option(..., "--question", "-q", help="Your question about this domain"),
    domain_id: str = typer.Option(..., "--domain", "-d", help="Domain ID to query"),
    workspace: Path = typer.Option(Path.cwd(), "--workspace", "-w"),

    json_output: bool = typer.Option(False, "--json", "-j"),

        typer.echo("ERROR: Capability 'ask.domain' not found. Ensure Phase 5 is complete.")

        workspace_path=str(workspace),
        domain_id=domain_id,
        question=question,
        max_cards=max_cards,
    )
    _display_result(result, json_output)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User question → LLM API | User-supplied question string flows through LangGraph to Anthropic API. Question is never interpolated into system prompt (always in HumanMessage). |
| Card content → LLM context | Workspace card content is loaded from filesystem and passed to LLM as context. Content stays within the requested domain_id (enforced by filter_by_domain node). |
| LLM output → CLI stdout | LLM-synthesized answer is rendered to user via CLI. Citations are validated against the card set before display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Spoofing | `llm_synthesize` node — prompt injection via user question | mitigate | System prompt explicitly restricts LLM to use ONLY provided card context. No tool access granted (no bind_tools). User input goes in HumanMessage, never system prompt per AI-SPEC §4b. |
| T-05-04 | Repudiation | `extract_citations` node — LLM cites nonexistent card IDs | mitigate | extract_citations validates every cited ID against the retrieved card set. Missing IDs are dropped with a structured warning (AI-SPEC §6 online guardrail: unverifiable citation detection). |
| T-05-05 | Information Disclosure | `filter_by_domain` node — content leak across domains | mitigate | filter_by_domain runs before build_context and llm_synthesize. Only cards matching `domain_id` reach the LLM. Structural enforcement per AI-SPEC §6 "Domain boundary enforcement" guardrail. |
| T-05-06 | Denial of Service | `build_context` → `llm_synthesize` — unbounded context / LLM cost | mitigate | max_cards capped at 50 (default 20). Card body truncated to 2000 chars.
