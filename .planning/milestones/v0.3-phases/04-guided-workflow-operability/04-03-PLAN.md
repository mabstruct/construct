| Skill procedure → MCP invocation | Skill documents specify MCP(connect) tools invoked by the agent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | T (Tampering) | CLI commands in skill docs | accept | Skill documents are markdown — not executable code. The agent reads them as procedures and runs CLI commands as typed. No shell injection vector through the document format. |
| T-04-03-02 | E (Elevation) | MCP tool invocation from skill | mitigate | MCP tools implement their own validation at the capability handler level. The skill document is instructional — the tool invocation goes through the same `cap.handler(**inputs)` dispatch as CLI commands. |
| T-04-03-03 | I (Info Disclosure) | Skill document contents | accept | Skill markdown files are local to the workspace. No sensitive data in procedure descriptions. |
