    ```python
    """Tag/keyword extraction pipeline — extract candidates from refs, write reviewable results."""
    from __future__ import annotations

    import json
    import re
    from datetime import datetime, timezone
    from pathlib import Path
    from dataclasses import dataclass, field
    from typing import Optional


    @dataclass
    class TagCandidate:
        id: str                             # auto-generated unique ID
        tag: str                            # the candidate tag/keyword
        domain_id: str | None = None        # source domain
        source_ref: str | None = None       # source ref filename
        confidence: float = 0.0             # 0.0 to 1.0
        frequency: int = 1                  # how many times this tag appeared
        status: str = "pending"             # pending | approved | rejected
        created_at: str = ""                # ISO timestamp
        notes: str = ""                     # extraction notes


    @dataclass
    class TagExtractionResult:
        success: bool
        candidates: list[TagCandidate]
        total_candidates: int
        new_candidates: int                 # not previously seen
        existing_seeds_skipped: int         # already in search-seeds.json
        error: str | None = None
    ```

    Core function: **extract_candidates(workspace: Path) -> TagExtractionResult**

    1. **Read refs** — Scan `workspace/refs/*.json` files
       - Load each ref JSON
       - Extract: `title`, `abstract` (from `abstract` field), `notes` (from `abstract` field — existing pattern), `domain_id`
       - Skip refs without abstract/notes content

    2. **Extract candidate tags** using a hybrid approach per discretion:
       - **LLM-assisted path**: For each ref, extract noun phrases and key concepts from abstract/notes
         - Use simple NLP heuristics (split by common delimiters, filter stop words, extract capitalized phrases)
         - Score by frequency across refs
       - **Fallback**: regex extraction of capitalized multi-word phrases (potential proper nouns, concepts)
       - **Dedup**: Normalize to lowercase, strip surrounding whitespace, collapse duplicates

    3. **Filter against existing search-seeds.json**:
       - Read `workspace/search-seeds.json`
       - Skip candidates that already match existing seed patterns (fuzzy match — case-insensitive substring)
       - Count skipped as `existing_seeds_skipped`

    4. **Load existing tag-candidates.json** (if exists) to detect truly new candidates
       - Compare new candidates against previously generated ones
       - Only count truly novel tags as `new_candidates`

    5. **Write candidates** to `workspace/log/tag-candidates.json`:
       - Append new candidates to existing ones (don't overwrite previous approvals)
       - Format: `{"candidates": [...], "generated_at": "...", "total": N}`

    6. **Return** TagExtractionResult with stats

    Error handling:
    - If `workspace/refs/` doesn't exist: return `TagExtractionResult(success=False, error="No refs directory found")`
    - If ref JSON has parse errors: skip that file, note in result
    - If `log/tag-candidates.json` is corrupted (invalid JSON): start fresh, log warning

    Helper: **load_existing_candidates(workspace) -> list[dict]** — reads existing tag-candidates.json
    Helper: **load_search_seeds(workspace) -> list[str]** — reads search-seeds, returns seed patterns
    Helper: **normalize_tag(text: str) -> str** — lowercase, strip, collapse whitespace
    Helper: **extract_phrases(text: str) -> list[str]** — extract candidate noun phrases from text
  </action>
