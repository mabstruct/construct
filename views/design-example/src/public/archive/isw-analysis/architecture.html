Section 1: Three-Phase Cycle

INITIATE

Human seeds material → Agent parses, extracts, proposes topics, builds taxonomy, embeds vectors

→

PROCESS

Daily autonomous cycle → Search (weighted tags + semantic neighbors + gap areas), Filter (relevance scoring), Extract (JSON), Connect (vector similarity), Compile (daily digest)

→

ADAPT

Human + Watson review → Qualify findings (accept/reject/defer), Reweight taxonomy, Redirect search patterns

→
🔄 Flywheel Concept: Each review cycle makes the next search more precise. The taxonomy evolves, weights adjust, and search patterns sharpen with every iteration. What begins as broad exploration becomes increasingly targeted discovery.

Section 2: Data Model

Three-Layer Model

Topics
Regions on the map — broad thematic areas
Tags
Landmarks within regions, with weights — refined search dimensions
Connections
Roads between entries — relationship types: extends, contradicts, parallels, grounds, cites, relates

Sample Entry Structure

{ "id": "entry_2025_001", "title": "ISW Military Assessment", "topics": ["military", "strategy"], "tags": { "conflict": 0.95, "analysis": 0.87, "geopolitics": 0.76 }, "source": "Brave Search API", "date": "2025-03-21", "connections": [ { "type": "extends", "ref": "entry_2025_000" } ], "embedding": [0.123, ..., 0.456] }

Section 5: Design Principles

1

Map, not Archive

Store extracted knowledge, not original files. Curate pathways through the landscape. Build relationships, not collections.

2

Domain-Agnostic

ISW first, next domain slots in with the same structure. Scalable architecture ready for law, finance, medicine, or any knowledge domain.

3

Human-in-the-Loop

Agent discovers, humans qualify. No auto-promotion of findings. Every acceptance, rejection, or deferral is a learning signal.

4

Transparent

Every search, decision, and rejection is logged. Audit trails enable reflection. Traceability builds trust and enables improvement.

5

Evolving

Taxonomy and topic list are living documents. Weights shift. Search patterns adapt. The system learns from each cycle.

6

References Over Copies

Summaries, extracts, links. Don't recreate the internet. Respect original sources while building your own knowledge graph.

Section 6: Multi-Agent System

Research Agent (RA)

Daily search, extraction, and digest compilation. Executes weighted searches, filters by relevance, connects entries, produces daily output.

Curator

Presents digest to user, collects feedback, facilitates reflection. Acts as intermediary between autonomous discovery and human judgment.

Watson (Main)

Orchestration, reflection, and adaptation proposals. Analyzes feedback, reweights taxonomy, redirects search patterns for next cycle.

Agent Loop Flow:

RA searches (with updated weights) → Curator presents findings to human → Human reflects and provides feedback → Watson analyzes feedback and proposes taxonomy/weight adjustments → RA executes next search with refined parameters (improved targeting, higher precision)
