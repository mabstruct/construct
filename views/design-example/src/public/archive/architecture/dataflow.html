File Lifecycle

The journey of a research result from discovery to knowledge graph integration.

Step 1: Brave Search Query
Search engine executes query from active seeds
Sources: search-seeds.json, external API
Step 2: Generate digest.md
Curator organizes results into daily markdown digest
Output: digests/YYYY-MM-DD.md with formatted results
Step 3: Check seen-urls.json
Deduplicate against previously seen sources
Updates: seen-urls.json with URLs from current digest
Step 4: Run digest_ingest.py
Parse markdown and extract structured data
Converts: digest.md → JSON objects with metadata
Step 5: Store in refs/digest/
Save extracted JSON entries to knowledge directory
Files: refs/digest/YYYY-MM-DD/*.json (one per result)
Step 6: Artifact Enrichment
Watson analyzes content for semantic connections
Adds: embeddings, tags, related concepts
Step 7: Human Review & Feedback
Human curator reviews digest and provides signals
Records: feedback-log.json with quality signals
Step 8: Taxonomy Update
Update topic hierarchy and cluster weights
Updates: taxonomy.json, search-seeds.json weights

Data Retention Policy

How CONSTRUCT manages data lifecycle to balance accessibility with storage efficiency.

Original Files (Raw Data)
PDF documents, web pages, and raw API responses are removed after extraction and enrichment. These temporary files are not retained long-term.
Extracted JSON (Permanent)
Structured extractions stored in refs/ directories are kept permanently. These are the canonical representation of each research result.
Vector Embeddings (Permanent)
ChromaDB stores all embeddings indefinitely for semantic search capability. Vectors enable cross-domain connections and discovery.
Daily Digests (Permanent)
Markdown digests are archived permanently in digests/ for historical reference and pattern analysis across time.
Git History (Permanent)
All changes to JSON files and taxonomy are tracked in git for version control and audit trail purposes.
Execution Logs (1 Year)
System logs in logs/ directory are kept for debugging purposes, typically retained for 12 months then archived.
